첫 화면에서 바로 코스와 인증 흐름을 이해하게 만드는 구조

사용자는 지도를 열자마자 어떤 챌린지가 어디에서 열리고, 어떤 난이도이며, 어디서 인증해야 하는지까지 한 번에 파악할 수 있습니다.

01

지도 중심 탐색

코스, 인증 지점, 권역 필터를 한 화면에 묶어 운영자와 참여자 모두가 빠르게 탐색할 수 있도록 설계했습니다.

02

챌린지 상태 즉시 확인

참가자 수, 완주 수, 난이도, 인증 방식이 선택된 카드와 지도 마커에 동시에 반영되어 운영 판단이 쉬워집니다.

03

전북 항공사진 강조

전라북도 지역만 항공사진 레이어를 올려 예산과 성능을 아끼면서도 핵심 운영 권역의 현장감을 살릴 수 있습니다.

Operational flow

실제 운영 흐름에 맞춘 메인 페이지 활용 방식

운영자

권역별 챌린지 배치 확인

지도에서 전국 코스를 비교하고, 전북 권역은 항공사진으로 도로 환경과 인증 지점 배치까지 빠르게 점검합니다.

참가자

코스 선택 후 즉시 참가

검색 또는 권역 탭으로 코스를 찾고, 거리·난이도·인증 조건을 확인한 뒤 바로 참여 페이지로 연결할 수 있는 구조입니다.

데이터팀

인증/랭킹 데이터 결합

인증 로그, 완주율, 랭킹 API를 연결하면 선택 카드와 지도 마커 상태를 실시간에 가깝게 업데이트할 수 있습니다.

FAQ

메인 홈페이지 구성 시 자주 묻는 질문

항공사진을 전국이 아니라 전북만 표시해도 되나요?

가능합니다. 이번 시안처럼 기본지도는 전국으로 유지하고, 전북 경계 안에서만 위성 타일을 오버레이해도 시각적으로 충분히 구분됩니다.

실제 챌린지 데이터를 붙이려면 무엇이 필요하나요?

코스 GeoJSON 또는 GPX, 인증 지점 좌표, 참가/완주 집계 API만 있으면 현재 구조에 바로 연결할 수 있습니다.

메인 화면에서 랭킹이나 코스 상세까지 확장할 수 있나요?

가능합니다. 현재 카드 영역은 랭킹 모달, 코스 상세 패널, 참가 CTA를 붙이기 좋도록 여유 있게 설계했습니다.
